VirtualBox Installation

Wasta-Linux VirutalBox Installation

In general, Wasta-Linux runs fine in VirtualBox after the VirtualBox Guest Additions have been installed on the Wasta-Linux guest virtual machine and settings are made correctly.  Before the VirtualBox Guest Additions are installed, it will not be possible to enable "3D Graphics", which Cinnamon requires, and you will get a warning message when logging in that you are in "Software Rendering Mode".  There are a few quirks to getting these "Guest Additions" installed that are documented here.

All versions:

  • Choose "Ubuntu" as the Linux version when installing Wasta-Linux in VirtualBox.
  • Ensure that 3D Graphics are enabled in the VirtualBox settings (see screen shot):

Wasta-Linux 14.04: A bug exists with the VirtualBox Guest Additions in versions of VirtualBox prior to 3.13.  It seems the bug is solved in the 3.13 (and newer) versions of VirtualBox.  So, if your version is older than that, even if you think you installed the Guest Additions it will not enable the 3D Graphics and you will get warning messages about Cinnamon running in "Software Rendering Mode".  See this post on the Linux Mint forums for more information.  Unlike Wasta-Linux 12.04, it seems that Wasta-Linux 14.04 will run fine in this "Software Rendering Mode", but it will require more CPU and thus appear to run "slow".  Remember that these are VirtualBox issues, and that a Wasta-Linux "real installation" will be fast and responsive in comparison!

Wasta-Linux 12.04: If you are only testing Wasta-Linux 12.04 in VirtualBox, then Wasta-Linux 12.04.0 already includes the VirtualBox package to enable 3D graphics, so it will work "out of the box" in VirtualBox!

However, Due to a conflict, Wasta-Linux 12.04.5 cannot include this package, so the guest additions must be installed manually. When attempting to install Wasta-Linux 12.04.5 in VirtualBox, this is the one situation where you should choose "install - start the installer directly" rather than "live - boot the Live System" from the boot menu.  The reason for this is that you won't be able to install the VirtualBox Guest Additions on this "Live Session".  In Wasta-Linux 12.04.5, rather than just using more CPU when Cinnamon is in "Software Rendering Mode" (like in Wasta-Linux 14.04), instead the Live System is practically unusable, with the screen flashing black, not clearing closed windows, etc.  So, again, please choose install directly rather than booting the Live System.

After the installation, these same screen problems and warnings about "Software Rendering Mode" will continue until the VirtualBox Guest Additions are installed.

To install the Guest Additions, choose "Devices | Install Guest Additions" (or "Host + D") from the VirutalBox menu.  See image below of the "software rendering mode" and menu selection:

Then it will ask for an administrative password (the password you created for the user at install time has administrative permissions, so use this password).  A terminal window will launch, warning you that you have an older version of Guest Additions installed, what do you want to do?

See image below of terminal window (and screen garbling due to 3D graphics not working correctly):

Type "yes" at this prompt and then <Enter> to continue and the Virtualbox Guest Additions should install successfully.  You will need to shutdown and restart for 3D graphics to be enabled.  Then when logging into Cinnamon, you should NOT get the "Software Rendering" message.

If the screen garbling is too difficult to deal with in when trying to install the Virtualbox Guest Additions, at the login window you can click the turquoise Wasta-Linux icon next to login name and choose "Gnome CLASSIC - no effects".  Then when you login, the (2D) Gnome Classic Panel will load.  There should be no screen garbling and you should be able to install the Virtualbox Guest Additions more easily.  Then, after successfully installing, restart the virtual machine and again at the login screen click the icon to choose Cinnamon again instead of Gnome Classic.
